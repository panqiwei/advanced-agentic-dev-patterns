<Band tone="tan" grow>
  <CoreDef>
    "Attention is a scarce budget — every token in the input spends the model's cognitive bandwidth. Context engineering is the craft of finding the smallest high-signal token set that completes the task."
  </CoreDef>
  <p class="tan-sub">—— four engineering moves against context rot ↓</p>
</Band>

<Band tone="paper-sunk" cols={2}>
  <div class="tech">
    <div class="tech__num">①</div>
    <Kicker>JUST-IN-TIME</Kicker>
    <p>Load on demand, never upfront. Tools, docs, memory surface only when the model actually needs them — attention stays sharp.</p>
  </div>
  <div class="tech">
    <div class="tech__num">②</div>
    <Kicker>COMPACTION</Kicker>
    <p>Summarize old turns. Keep structural conclusions, drop redundant chatter; long conversations stop growing linearly.</p>
  </div>
  <div class="tech">
    <div class="tech__num">③</div>
    <Kicker>NOTE-TAKING</Kicker>
    <p>Agent keeps its own external notes. Cross-session recall + working memory separated from episodic memory.</p>
  </div>
  <div class="tech">
    <div class="tech__num">④</div>
    <Kicker>SUB-AGENT</Kicker>
    <p>Split long tasks across isolated contexts. The parent consumes the child's conclusions, never its trace.</p>
  </div>
</Band>
