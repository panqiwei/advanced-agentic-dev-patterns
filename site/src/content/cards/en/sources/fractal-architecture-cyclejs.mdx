<Band tone="paper-raised" dense>
  <h2 class="card-title">Fractal Architecture · the common substrate of front-end componentization</h2>
  <p class="card-subtitle">Self-similarity + uniform API = any subtree is itself a valid tree</p>
</Band>

<Band tone="tan" dense>
  <CoreDef>If you understand how one component works, you <span class="em">understand all components</span> — because all components expose the same API shape, and every subtree is a valid, complete tree.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="rules">
    <div class="rl rl--1">
      <div class="rl-n">Ⅰ</div>
      <div class="rl-body">
        <div class="rl-k">An app is a tree of components with the same API</div>
        <div class="rl-v">All components expose a uniform interface — the <span class="em">self-similarity condition</span> of a fractal</div>
      </div>
    </div>
    <div class="rl rl--2">
      <div class="rl-n">Ⅱ</div>
      <div class="rl-body">
        <div class="rl-k">Each component can contain other components</div>
        <div class="rl-v">Composition is recursive — any component can be embedded in any context</div>
      </div>
    </div>
    <div class="rl rl--3">
      <div class="rl-n">Ⅲ</div>
      <div class="rl-body">
        <div class="rl-k">The top-level component is not essentially different from any other</div>
        <div class="rl-v">The root has no special privilege — a component doesn't know whether it is "root" or "leaf"</div>
      </div>
    </div>
    <div class="rl rl--4">
      <div class="rl-n">Ⅳ</div>
      <div class="rl-body">
        <div class="rl-k">Glue code is separated from application logic</div>
        <div class="rl-v">Bootstrap / wiring happens outside the component hierarchy — the precondition for fractal architecture to hold</div>
      </div>
    </div>
  </div>
</Band>

<Band tone="paper-sunk" dense>
  <div class="frameworks">
    <div class="fw"><span class="fw-k">React / Angular</span><span class="fw-v">XML-style declarative components · bootstrap API</span></div>
    <div class="fw"><span class="fw-k">Elm</span><span class="fw-v"><span class="mono">init</span>/<span class="mono">update</span>/<span class="mono">view</span> three functions + <span class="mono">main</span> passes in the root</span></div>
    <div class="fw"><span class="fw-k">Cycle.js</span><span class="fw-v">Pure functions observables → observables · separate wiring layer</span></div>
  </div>
</Band>
