<Band tone="paper-raised" dense>
  <h2 class="card-title">Harness design for long-running agents</h2>
  <p class="card-subtitle">When a task exceeds one context window — two-agent architecture + externalized state handoff</p>
</Band>

<Band tone="tan">
  <CoreDef>Two failure modes, one root cause: one-shotting (context burned on a half-finished artifact) and premature victory (mistakenly declaring "done"). The fix is to trade "finish in one shot" for "incremental progress + state handoff" — Initializer builds the scaffold, Coder advances feature by feature.</CoreDef>
</Band>

<Band tone="paper-raised" cols={2} grow>
  <div class="agent">
    <div class="agent__label">① INITIALIZER</div>
    <div class="agent__role">one-time</div>
    <ul class="agent__list">
      <li>Write <span class="mono">init.sh</span> (dev env bootstrap)</li>
      <li>Create <span class="mono">claude-progress.txt</span></li>
      <li>Generate JSON feature list (200+ entries)</li>
      <li>Initial git commit</li>
    </ul>
  </div>
  <div class="agent agent--coder">
    <div class="agent__label">② CODER</div>
    <div class="agent__role">per session</div>
    <ul class="agent__list">
      <li>Read progress + git log to recover state</li>
      <li>Pick one feature, implement end-to-end</li>
      <li>Verify via Puppeteer MCP</li>
      <li>Commit + update <span class="mono">passes</span> field</li>
    </ul>
  </div>
</Band>

<Band tone="paper-sunk" dense>
  <div class="handoff">
    <span class="hf-k">Handoff artifacts</span>
    <span class="hf-v"><span class="mono">init.sh</span> · <span class="mono">claude-progress.txt</span> · <span class="mono">features.json</span> · <span class="mono">git log</span></span>
  </div>
  <div class="handoff">
    <span class="hf-k">Key constraint</span>
    <span class="hf-v">Agent can only edit <span class="mono">passes</span> — JSON resists "drive-by edits" better than Markdown</span>
  </div>
</Band>
