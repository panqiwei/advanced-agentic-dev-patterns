<Band tone="paper-raised" dense>
  <h2 class="card-title">Building Agents with the Claude Agent SDK</h2>
  <p class="card-subtitle">Thariq Shihipar (Anthropic) — the Claude Agent SDK design philosophy: give the agent a computer</p>
</Band>

<Band tone="tan">
  <CoreDef>Core design principle: "Give Claude a Computer" — whatever tools a programmer needs, Claude needs too. The agent runs in a gather context → take action → verify work → repeat feedback loop — not a predefined graph, but a model-driven implicit loop architecture where the model itself decides the next step.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="three-layers">
    <Kicker>Three agent capability layers</Kicker>
    <div class="tl-row tl-gather">
      <span class="tl-name">1. Gather Context</span>
      <div class="tl-items">
        <span class="ti">Agentic search: filesystem structure + bash (folder layout is itself a form of context engineering)</span>
        <span class="ti">Subagents: parallelism + context isolation — only relevant info returns to the orchestrator</span>
        <span class="ti">Compaction: auto-compress history as context approaches the ceiling</span>
      </div>
    </div>
    <div class="tl-row tl-act">
      <span class="tl-name">2. Take Action</span>
      <div class="tl-items">
        <span class="ti">Tools: the agent's primary execution primitives — they occupy prime context real estate and shape model decisions</span>
        <span class="ti">Bash &amp; scripts + code generation + MCP for external service integration</span>
      </div>
    </div>
    <div class="tl-row tl-verify">
      <span class="tl-name">3. Verify Work</span>
      <div class="tl-items">
        <span class="ti">Rules-based (lint) → visual feedback (multimodal screenshots) → LLM-as-judge</span>
      </div>
    </div>
  </div>
  <div class="design">
    <Kicker>Design principles</Kicker>
    <KV label="Semantic vs Agentic Search">Semantic retrieval is fast but opaque; reach for agentic search first — more accurate, cheaper to maintain</KV>
    <KV label="Tools as documentation">Define tools the way you would write docs for a junior developer — ACI matters as much as HCI</KV>
  </div>
</Band>
