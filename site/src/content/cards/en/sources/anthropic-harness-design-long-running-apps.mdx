  <h2 class="card-title">Harness Design for Long-Running App Development</h2>
  <p class="card-subtitle">A GAN-inspired Generator-Evaluator architecture — split generation from evaluation to defeat the "self-eval optimism bias"</p>
</Band>

<Band tone="tan">
  <CoreDef>Three-agent architecture: <span class="hl">Planner</span> (1-4 sentence prompt → 200+ feature spec), <span class="hl">Generator</span> (implements sprint by sprint + self-eval), <span class="hl">Evaluator</span> (actually operates the app via Playwright MCP). Core insight: an agent evaluating itself is naturally optimistic; split the evaluator off and it's far easier to tune into a strict critic.</CoreDef>
</Band>

    <span class="ins-k">Method</span>
    <span class="ins-v">Every harness component encodes an assumption about "what the model can't do" — re-test and prune as models improve. Planner always pays off; Evaluator matters when the task sits at the capability frontier; sprint structure became unnecessary after Opus 4.6.</span>
