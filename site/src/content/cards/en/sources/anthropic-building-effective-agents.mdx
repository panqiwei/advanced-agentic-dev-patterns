  <h2 class="card-title">Building Effective AI Agents</h2>
  <p class="card-subtitle">Schluntz &amp; Zhang (Anthropic, 2024-12-20) — lessons distilled from dozens of customer agent builds</p>
</Band>

<Band tone="tan">
  <CoreDef>Core claim: the most successful implementations don't lean on heavy frameworks — they use simple, composable patterns. The base building block is the "augmented LLM": retrieval + tools + memory, with standardized interfaces via protocols like MCP. A single LLM call + retrieval + in-context examples is usually enough — only add complexity when there's evidence it improves outcomes.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="patterns">
    <Kicker>Five workflows + one agent pattern</Kicker>
    <div class="pt-row pt-chain">
      <span class="pr-name">Prompt Chaining</span>
      <span class="pr-desc">Task decomposed into sequential steps; gates optional between them</span>
    </div>
    <div class="pt-row pt-route">
      <span class="pr-name">Routing</span>
      <span class="pr-desc">Classify input, then dispatch to specialized handlers — semantic routing</span>
    </div>
    <div class="pt-row pt-para">
      <span class="pr-name">Parallelization</span>
      <span class="pr-desc">Sectioning (split work) + voting (ensemble of answers)</span>
    </div>
    <div class="pt-row pt-orch">
      <span class="pr-name">Orchestrator-Workers</span>
      <span class="pr-desc">Central LLM dynamically decomposes the task, dispatches to workers, synthesizes results</span>
    </div>
    <div class="pt-row pt-eval">
      <span class="pr-name">Evaluator-Optimizer</span>
      <span class="pr-desc">Generate-evaluate iteration — feedback-driven refinement</span>
    </div>
    <div class="pt-row pt-agent">
      <span class="pr-name">Agent (autonomous)</span>
      <span class="pr-desc">LLM dynamically directs its own flow — open-ended problems with unpredictable step counts</span>
    </div>
  </div>
  <div class="aci">
    <Kicker>ACI design principle</Kicker>
    <KV label="ACI matters as much as HCI">Write tool definitions with the clarity you'd use in docs for a junior developer — the agent is the primary user of the tool</KV>
  </div>
</Band>
