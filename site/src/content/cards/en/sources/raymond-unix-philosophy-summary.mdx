      <div class="bq-label">The Unix designer's challenge</div>
      <div class="bq-text">
        How do you design a system that accommodates <span class="em">programs as yet unthought of</span>?
      </div>
      <div class="bq-answer">
        → "everything is a file" unifies all I/O through <span class="mono">read()</span> / <span class="mono">write()</span>
      </div>
    </div>
    <div class="bq-div">↕</div>
    <div class="bq-new">
      <div class="bq-label">The meta-harness challenge</div>
      <div class="bq-text">
        How do you design an agent platform that accommodates <span class="em">harnesses as yet unimplemented</span>?
      </div>
      <div class="bq-answer">
        → unify all tool invocations through <span class="mono">execute(name, input) → string</span>
      </div>
    </div>
  </div>
</Band>

<Band tone="paper-sunk">
  <CoreDef>
    Shared design philosophy — <span class="hl">maximize applicability by minimizing the interface</span>. Both give up on the ambition to "anticipate every possible component" and instead define a <span class="em">minimal universal contract</span>, letting future developments evolve freely within it.
  </CoreDef>
</Band>
