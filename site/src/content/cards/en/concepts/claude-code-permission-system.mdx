  <h2 class="card-title">Claude Code Permission System</h2>
  <p class="card-subtitle">Claude Code Permission System — tool approval tiered by reversibility</p>
</Band>

<Band tone="tan">
  <CoreDef>The design logic behind the three tiers is reversibility. Read-only operations have no side effects and run freely. File edits are reversible (git), so session-scoped approval suffices. Bash can execute arbitrary system operations, so it needs a permanent record.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="tier-stack">
    <div class="tier t-read">
      <span class="tier-name">Read-only tools</span>
      <span class="tier-examples">Read · Grep · Glob</span>
      <span class="tier-policy">no approval required</span>
    </div>
    <div class="tier t-write">
      <span class="tier-name">File-editing tools</span>
      <span class="tier-examples">Edit · Write</span>
      <span class="tier-policy">"don't ask again" → session-scoped allow</span>
    </div>
    <div class="tier t-bash">
      <span class="tier-name">Bash commands</span>
      <span class="tier-examples">shell execution (arbitrary system ops)</span>
      <span class="tier-policy">"don't ask again" → saved as permanent allow rule</span>
    </div>
  </div>
  <div class="rule-syntax">
    <Kicker>rule syntax examples</Kicker>
    <div class="rule-examples">
      <code>allow: ["Bash(npm run *)", "WebFetch(domain:github.com)"]</code>
      <code>deny: ["Bash(git push *)", "Read(~/.ssh/**)"]</code>
    </div>
  </div>
</Band>
