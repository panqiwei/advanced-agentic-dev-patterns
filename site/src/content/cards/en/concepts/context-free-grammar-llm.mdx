<Band tone="paper-raised" dense>
  <h2 class="card-title">Context-Free Grammar (LLM)</h2>
  <p class="card-subtitle">Context-Free Grammar — the grammatical basis of JSON Schema constrained decoding</p>
</Band>

<Band tone="tan">
  <CoreDef>JSON is a context-free language (CFG-recognizable), not a regular one (FSM-recognizable). This theoretical distinction has real consequences: FSMs can't natively handle recursively nested JSON structures — CFGs can.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="chomsky-hierarchy">
    <Kicker>Chomsky hierarchy</Kicker>
    <div class="ch-stack">
      <div class="ch-level cl-re">
        <span class="cl-name">recursively enumerable</span>
        <span class="cl-machine">Turing machine</span>
      </div>
      <div class="ch-level cl-cs">
        <span class="cl-name">context-sensitive</span>
        <span class="cl-machine">linear-bounded automaton</span>
      </div>
      <div class="ch-level cl-cf active">
        <span class="cl-name">context-free (CFG)</span>
        <span class="cl-machine">pushdown automaton · JSON ✓</span>
      </div>
      <div class="ch-level cl-reg">
        <span class="cl-name">regular (FSM)</span>
        <span class="cl-machine">finite-state machine · no nesting ✗</span>
      </div>
    </div>
  </div>
  <div class="workflow">
    <Kicker>practical workflow</Kicker>
    <div class="wf-flow">
      <span class="wf-step">JSON Schema</span>

      <span class="wf-step">dynamic mask at inference</span>

  </div>
</Band>

<Band tone="paper-sunk" dense>
  <p class="advantage-note">CFG's real edge over FSM: a nested object <code>{"{field: {nested: ...}}"}</code> needs the stack memory of a pushdown automaton to track nesting depth — regular languages can't express this recursive structure.</p>
</Band>
