  <h2 class="card-title">SSM Hybrid Architecture</h2>
  <p class="card-subtitle">SSM Hybrid Architecture — fuse state-space models with Transformers to solve long-context linear-complexity</p>
</Band>

<Band tone="tan">
  <CoreDef>Pure Transformers are O(n²) in long context — a hard bottleneck for agent tasks. SSM hybrids swap most attention layers with selective state-space models (Mamba) while keeping a few attention layers for global reasoning. Jamba: 1/8 attention + 7/8 Mamba + MoE, enabling 256K-token context, linear complexity, and lower inference latency.</CoreDef>
</Band>

    <Kicker>Implications for agent design</Kicker>
    <KV label="Mamba-3 inference-first">Inference 2–8× faster than Transformer — suits high-frequency harness calls</KV>
    <KV label="KV-cache compression">SSM state is a fixed-size recurrent buffer, not growing with length — eases VRAM pressure</KV>
    <KV label="Long-context agents">A 256K window makes single-pass processing of large codebases feasible, reducing chunking complexity</KV>
    <KV label="Current limits">Training efficiency still trails pure Transformers; attention layers remain critical for precise retrieval tasks</KV>
