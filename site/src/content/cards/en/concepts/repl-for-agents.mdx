<Band tone="paper-raised" dense>
  <h2 class="card-title">REPL for Agents</h2>
  <p class="card-subtitle">REPL for Agents — enter a stateful session; context accumulates, consecutive ops need no restart</p>
</Band>

<Band tone="tan">
  <CoreDef>Subcommand mode pays startup cost every call (process, file I/O, serialization). REPL mode keeps state in memory, so once the agent enters a session, operations are nearly free. Fits stateful workflows (image editing, code iteration) and exploratory tasks that need undo/redo.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="mode-compare">
    <Kicker>Three interface modes compared</Kicker>
    <div class="mc-row mc-header"><span>Dimension</span><span>Subcommand</span><span>REPL</span><span>MCP Tool</span></div>
    <div class="mc-row"><span>State</span><span>Externalized (files)</span><span>Resident process</span><span>Resident server</span></div>
    <div class="mc-row"><span>Startup cost</span><span>Every call</span><span>Once on entry</span><span>Once on connect</span></div>
    <div class="mc-row"><span>Best for</span><span>Single-step, scripted</span><span>Multi-step, trial-and-error</span><span>Cross-process, permissions</span></div>
  </div>
  <div class="repl-principles">
    <Kicker>Four REPL design principles</Kicker>
    <KV label="Dual mode">Bare command enters REPL; with args runs once — one binary serves both cases</KV>
    <KV label="Self-describing banner">On start, print banner: tool name + absolute SKILL.md path + current session state</KV>
    <KV label="State in prompt">{"[gimp:poster.xcf *] > — project + dirty flag + undo stack; no separate status query needed"}</KV>
    <KV label="Serializable session">On exit, dump JSON session file; on next start, load it back — efficient and persistent</KV>
  </div>
</Band>
