<Band tone="paper-raised" dense>
  <h2 class="card-title">Fractal Architecture</h2>
  <p class="card-subtitle">Fractal Architecture — self-similarity: any subtree is a valid tree</p>
</Band>

<Band tone="tan">
  <CoreDef>Every component in the system shares the same structure and interface as the whole. Composability doesn't come from components being small — it comes from interface consistency. When interfaces are isomorphic, the composed result satisfies the same API contract, and composition recurses.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="four-rules">
    <Kicker>four rules (Telesh 2016, distilled from React / Angular / Elm / Cycle.js)</Kicker>
    <div class="rule-item">
      <span class="rule-n">①</span>
      <div>
        <span class="rule-name">uniform interface</span>
        <span class="rule-desc">the app is a tree of components sharing one API</span>
      </div>
    </div>
    <div class="rule-item">
      <span class="rule-n">②</span>
      <div>
        <span class="rule-name">recursive composition</span>
        <span class="rule-desc">every component can contain other components</span>
      </div>
    </div>
    <div class="rule-item">
      <span class="rule-n">③</span>
      <div>
        <span class="rule-name">no privileged root</span>
        <span class="rule-desc">top-level and leaf components are not fundamentally different</span>
      </div>
    </div>
    <div class="rule-item">
      <span class="rule-n">④</span>
      <div>
        <span class="rule-name">glue separated from logic</span>
        <span class="rule-desc">bootstrap / wiring code lives outside the component system; a component doesn't know its own position in the tree</span>
      </div>
    </div>
  </div>
  <div class="agent-mapping">
    <KV label="Agent Skills mapping">each skill is a directory with the same interface (SKILL.md + uniform protocol) — rules ①②</KV>
    <KV label="implicit-loop echo">the agent doesn't know the global path; behavior is shaped by constraints — the structural isomorphism of rules ③④</KV>
  </div>
</Band>
