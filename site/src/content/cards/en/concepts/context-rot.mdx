  <h2 class="card-title">Context Rot</h2>
  <p class="card-subtitle">Context Rot — systematic LLM decay as token count grows</p>
</Band>

<Band tone="tan">
  <CoreDef>Models don't process their context window uniformly — token #10,000 is less reliable than token #100. The task isn't harder; the same content just degrades systematically at greater context length.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <PillarGrid cols={3}>
    <Pillar label="Attention dilution">As input grows, per-token attention thins out; lower needle semantic similarity accelerates decay</Pillar>
    <Pillar label="Distractor interference">Relevant-but-wrong content interferes with retrieval; Claude tends to abstain, GPT to hallucinate — family-level divergence</Pillar>
    <Pillar label="Structural interference">A logically coherent haystack hurts more than a shuffled one — attention gets "pulled" by structure</Pillar>
  </PillarGrid>
</Band>

<Band tone="paper-sunk" dense>
  <div class="context-rot-note">
    <div class="crn-row">
      <span class="crn-label">vs Lost-in-Middle</span>
      <span class="crn-text">LITM is a positional effect (middle is weak); Context Rot is a global decay trend (longer is weaker) — both need context engineering to counter</span>
    </div>
    <div class="crn-row">
      <span class="crn-label">Engineering implication</span>
      <span class="crn-text">Not "if it fits, stuff it" — just-in-time loading and active compaction aren't just saving tokens, they're quality insurance</span>
    </div>
  </div>
