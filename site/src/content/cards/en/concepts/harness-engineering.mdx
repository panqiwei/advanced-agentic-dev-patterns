<Band tone="paper-raised" dense>
  <h2 class="card-title">Harness Engineering</h2>
  <p class="card-subtitle">Harness Engineering — sum of system prompt + tools + orchestration + state + permissions + feedback</p>
</Band>

<Band tone="tan">
  <CoreDef>A harness is not "glue code around an API call" — it is a complete control system. It does not restrict the agent; it lets the agent's capability land reliably. Agent failure signals an insufficient environment, not a model-capability ceiling. The right response: a better harness.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="harness-layers">
    <div class="hl-row"><span class="hl-label">System Prompt</span><span class="hl-desc">Global constraints · role definition · low-frequency component</span></div>
    <div class="hl-row"><span class="hl-label">Tool Defs</span><span class="hl-desc">Local capability boundaries · ACI design · declarative interface</span></div>
    <div class="hl-row"><span class="hl-label">Permissions</span><span class="hl-desc">Boundary control · high-frequency execution · risk-tiered</span></div>
    <div class="hl-row"><span class="hl-label">Feedback Loop</span><span class="hl-desc">End-to-end tests · Evaluator-Optimizer · MOP detection</span></div>
    <div class="hl-row"><span class="hl-label">State Mgmt</span><span class="hl-desc">Feature tracking · externalized progress · cross-session continuity</span></div>
    <div class="hl-row"><span class="hl-label">Termination</span><span class="hl-desc">Cap constraints · failure detection · human-in-the-loop triggers</span></div>
  </div>
  <div class="two-principles">
    <KV label="Anthropic principle">Every harness component encodes an assumption about "what the model can't do" — re-test it with each model upgrade; ask "what can I stop doing?"</KV>
    <KV label="OpenAI principle">Enforce invariants, not implementations — mechanically enforce architectural boundaries while allowing freedom within them</KV>
  </div>
</Band>
