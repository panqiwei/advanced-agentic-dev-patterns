  <h2 class="card-title">Neurosymbolic AI Taxonomy</h2>
  <p class="card-subtitle">Kautz's six types — a spectrum of neural-symbolic integration architectures from loosest to tightest coupling</p>
</Band>

<Band tone="tan">
  <CoreDef>Henry Kautz (AAAI 2020) proposed the most influential taxonomy of neurosymbolic architectures to date. Notation convention: brackets denote embedding, underscores denote wrapping. Core trade-off: tighter coupling = stronger verifiability + harder training.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="six-types">
    <div class="type-row t1"><span class="t-code">Type 1</span><span class="t-name">Symbolic_Neuro_Symbolic</span><span class="t-rep">Transformer NLP</span><span class="t-note">neural processing embedded inside symbolic boundaries — standard NLP paradigm</span></div>
    <div class="type-row t2"><span class="t-code">Type 2</span><span class="t-name">Symbolic[Neuro]</span><span class="t-rep">AlphaGo</span><span class="t-note">symbolic frame (MCTS) calls a neural subcomponent</span></div>
    <div class="type-row t3"><span class="t-code">Type 3</span><span class="t-name">Neuro;Symbolic</span><span class="t-rep">NS-CL</span><span class="t-note">neural perception + symbolic reasoning running side by side</span></div>
    <div class="type-row t4"><span class="t-code">Type 4</span><span class="t-name">Neuro:Symbolic→Neuro</span><span class="t-rep">LNN</span><span class="t-note">symbolic knowledge compiled into neural weights; inference is purely neural</span></div>
    <div class="type-row t5"><span class="t-code">Type 5</span><span class="t-name">Neuro_Symbolic</span><span class="t-rep">LTN</span><span class="t-note">logic rules as loss constraints; differentiable fusion</span></div>
    <div class="type-row t6"><span class="t-code">Type 6</span><span class="t-name">Neuro[Symbolic]</span><span class="t-rep">not yet realized</span><span class="t-note">symbolic reasoning embedded inside the neural net — full System 1+2 integration</span></div>
  </div>
  <div class="spectrum-note">
    <KV label="Coupling spectrum">Type 1 (loosest, easiest to train) → Type 6 (tightest, strongest verifiability)</KV>
    <KV label="Engineering reality">constrained decoding ≈ Type 1/4; LLM + KG ≈ Type 2/3; LIME post-hoc explanations ≠ any type</KV>
  </div>
</Band>
