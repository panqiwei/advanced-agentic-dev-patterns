<Band tone="paper-raised" dense>
  <h2 class="card-title">Permission Modes</h2>
  <p class="card-subtitle">Permission Modes — the global approval-policy switch that sets the default handling of every tool call</p>
</Band>

<Band tone="tan">
  <CoreDef>The permission mode sets the agent's "baseline" — specific allow/ask/deny rules stack on top. Six modes form an autonomy spectrum from most conservative to most aggressive. Core principle: deny-first semantics hold in every mode.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="mode-table">
    <div class="mt-row mt-header"><span>Mode</span><span>Behavior</span><span>Fits</span></div>
    <div class="mt-row m-dontask"><span class="m-name">dontAsk</span><span>auto-reject; only pre-authorized tools</span><span>high safety, minimal permission surface</span></div>
    <div class="mt-row m-default"><span class="m-name">default</span><span>prompt on first use</span><span>day-to-day development</span></div>
    <div class="mt-row m-plan"><span class="m-name">plan</span><span>read-only analysis; no mutating execution</span><span>planning / code review</span></div>
    <div class="mt-row m-accept"><span class="m-name">acceptEdits</span><span>auto-accept file edits</span><span>trusted file-modification scenarios</span></div>
    <div class="mt-row m-auto"><span class="m-name">auto</span><span>classifier auto-validates operation intent</span><span>trusted automation</span></div>
    <div class="mt-row m-bypass"><span class="m-name">bypassPermissions</span><span>skip most prompts</span><span>OS-level isolated container (dangerous)</span></div>
  </div>
  <div class="special-notes">
    <KV label="auto classifier">reads the autoMode config (environment/allow/soft_deny); allow/soft_deny fully replace default rules</KV>
    <KV label="Protected directories">even under bypass, .git/.claude/.vscode still prompt — guards against corrupting repo state</KV>
    <KV label="Org lockdown">managed settings can disable bypassPermissions / auto — the canonical scope-hierarchy use case</KV>
  </div>
</Band>
