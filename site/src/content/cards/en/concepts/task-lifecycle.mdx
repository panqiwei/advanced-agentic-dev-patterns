<Band tone="tan">
  <CoreDef>An A2A Task is the core unit of work: a unique ID + an explicit state transition path. After submission it enters submitted → working, then forks to four terminal states (completed / failed / canceled / rejected) or input-required (awaiting more information from a human). Once terminal, it accepts no new messages — sending to a completed Task returns UnsupportedOperationError.</CoreDef>
</Band>

    <KV label="Polling">POST sendMessage → GET getTask in a loop → fetch artifacts once completed. Fits simple tasks.</KV>
    <KV label="SSE streaming">One-way push: TaskStatusUpdateEvent + TaskArtifactUpdateEvent → live progress, fits long tasks.</KV>
