  <p class="card-subtitle">Agent-Native Software — agents as a first-class user type</p>
</Band>

<Band tone="tan">
  <CoreDef>Not: build a GUI first, then retrofit for agents. Agent I/O, state, and discovery needs diverge from human ones far enough to derive a different software shape — agent-native is the result of that derivation.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <PillarGrid cols={3}>
    <Pillar label="Structured-first">Every output toggles to JSON; errors are objects with code + remediation, not tracebacks</Pillar>
    <Pillar label="Self-describing">--help / schema / metadata answer "what can you do"; docs and code live in one tool</Pillar>
    <Pillar label="Composable">Each command does one thing, pipes cleanly; state is explicit, serializable as text</Pillar>
    <Pillar label="Explicit state">Long tasks save/load; undo primitives supported; never relies on session continuity</Pillar>
    <Pillar label="Idempotency">Side effects clearly marked; read vs write vs irreversible; failures retry cleanly</Pillar>
    <Pillar label="POSIX-first">Reuse what agents already know: shell / pip / env vars; avoid bespoke SDKs</Pillar>
  </PillarGrid>
</Band>

<Band tone="paper-sunk" dense>

    <span class="debate-b">Position B · New protocols needed: MCP/A2A handle cross-machine, permissions, bidirectional streams (ecosystem direction)</span>
