<Band tone="paper-raised" dense>
  <h2 class="card-title">Tool Design</h2>
  <p class="card-subtitle">Tool Design — the engineering practice of designing tool interfaces for LLM agents, on par with prompt engineering</p>
</Band>

<Band tone="tan">
  <CoreDef>Tool definitions and specs deserve the same prompt engineering effort as the overall prompt. The most common agent failure mode: a bloated toolset — overlapping coverage or ambiguous decision points. Golden rule: if a human engineer cannot determine with certainty which tool to use in a given situation, neither can the agent.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="design-principles">
    <Kicker>Core design principles</Kicker>
    <div class="dp-row">
      <span class="dp-label">Format choice</span>
      <span class="dp-body">Pick the format the model can most easily produce — close to formats naturally present in training data, give the model enough tokens to "think", and avoid format overhead (exact line counts, string escaping)</span>
    </div>
    <div class="dp-row">
      <span class="dp-label">Poka-yoke</span>
      <span class="dp-body">Make mistakes hard via parameter design, not prompt nagging — mistake-proof design beats warning text</span>
    </div>
    <div class="dp-row">
      <span class="dp-label">Minimal toolset</span>
      <span class="dp-body">Second-order benefits of a lean toolset: less choice ambiguity, and more reliable context maintenance and pruning</span>
    </div>
  </div>
  <div class="dual-pattern">
    <Kicker>CLI-Anything dual-mode dual-output</Kicker>
    <KV label="Dual interaction modes">Same binary: no-arg enters REPL (multi-step iteration) / with args acts as one-shot call — serves humans and agents at once</KV>
    <KV label="Dual output formats">Built-in --json flag toggles machine-readable structure vs human-readable tables — no extra SDK needed</KV>
    <KV label="Return-value efficiency">Verbose tool output directly consumes the model's attention budget — keep returns compact</KV>
  </div>
</Band>
