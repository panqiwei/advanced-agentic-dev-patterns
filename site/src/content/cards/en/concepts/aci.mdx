<Band tone="paper-raised" dense>
  <h2 class="card-title">ACI — Agent-Computer Interface</h2>
  <p class="card-subtitle">An interface discipline for models — invest as much design effort here as HCI gets</p>
</Band>

<Band tone="tan">
  <CoreDef>Put yourself in the model's shoes: given only the tool description and parameters, is the usage obvious? Spend ACI effort on par with HCI.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <PillarGrid cols={2}>
    <Pillar label="Self-describing names">Parameter names and descriptions make usage obvious — the "write docs for a junior dev" standard</Pillar>
    <Pillar label="Poka-yoke">Mistake-proof by design: force absolute paths over relative ones; eliminate error-prone arguments</Pillar>
    <Pillar label="Test + iterate">Run many samples, watch where the model fails, then change the interface — not the prompt</Pillar>
    <Pillar label="Infrastructure ACI">llms.txt / MCP / CLI-first — make the whole internet LLM-friendly</Pillar>
  </PillarGrid>
</Band>

<Band tone="paper-sunk" dense>
  <div class="insight-row">
    <span class="insight-label">Anthropic in practice</span>
    <span class="insight-text">On SWE-bench, they spent more time optimizing the tool interface than the prompt. Forcing absolute paths drove path errors to zero.</span>
  </div>
  <div class="insight-row">
    <span class="insight-label">Karpathy extends it</span>
    <span class="insight-text">Agents are the new consumers of information — meet LLMs halfway. llms.txt / Docs-for-LLMs / MCP are infrastructure-level ACI.</span>
  </div>
</Band>
