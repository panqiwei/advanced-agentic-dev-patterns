  <p class="card-subtitle">An agent's digital business card — the carrier of A2A capability discovery</p>

<Band tone="tan">
  <CoreDef>A client agent only needs the domain. GET /.well-known/agent-card discovers at runtime: what the other side can do, where to send requests, what auth is required, whether streaming is supported.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="card-schema">
    <Kicker>Agent Card core fields</Kicker>
    <KV label="name / description">Agent identity declaration</KV>
    <KV label="url">A2A service endpoint (sendMessage target)</KV>
    <KV label="capabilities">streaming · pushNotifications — boolean capability flags</KV>
    <KV label="skills">Fine-grained list of task types the agent handles</KV>
    <KV label="securitySchemes">OAuth2 / OpenID Connect / API Key requirements</KV>
    <KV label="extensions">Custom protocol extensions</KV>
  </div>
</Band>

<Band tone="paper-sunk" dense>
  <div class="flow-row">
    <span class="flow-step">GET /.well-known/agent-card</span>
    <span class="flow-arrow">→</span>
    <span class="flow-step">parse securitySchemes, fetch token</span>
    <span class="flow-arrow">→</span>
    <span class="flow-step">POST sendMessage to url</span>
  </div>
  <p class="note-text">capabilities is runtime capability negotiation — use SSE only when streaming: true; register a webhook only when pushNotifications: true.</p>
</Band>
