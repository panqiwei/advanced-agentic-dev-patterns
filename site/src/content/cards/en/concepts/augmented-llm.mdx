  <p class="card-subtitle">Augmented LLM — LLM + retrieval + tools + memory</p>

<Band tone="tan">
  <CoreDef>The building block of agentic systems. Current models actively wield their augmentations: generate their own search queries, pick tools, decide what to retain. Key: tailor to the use case — don't stack generic capabilities.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <PillarGrid cols={3}>
    <Pillar label="Retrieval">Load relevant context on demand; just-in-time beats upfront stacking; Claude Code = CLAUDE.md preload + glob/grep on demand</Pillar>
    <Pillar label="Tools">MCP-standardized interfaces; tool docs = ACI design; unpredictable tool behavior is the top source of agent failure</Pillar>
    <Pillar label="Memory">In-session context window; cross-session external store; memory overhead can outweigh the gain (see the episodic-memory lesson)</Pillar>
  </PillarGrid>
</Band>

<Band tone="paper-sunk" dense>
  <div class="karpathy-insight">
    <span class="ki-label">KARPATHY</span>
    <span class="ki-text">LLM = CPU (raw compute); augmented LLM = CPU + coprocessors/peripherals — retrieval is DMA, tools are I/O, memory is the cache hierarchy. But this CPU processes tokens, and its behavior is statistical — don't assume tool calls are idempotent.</span>
  </div>
</Band>
