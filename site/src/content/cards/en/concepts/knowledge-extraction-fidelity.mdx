<Band tone="paper-raised" dense>
  <h2 class="card-title">Knowledge Extraction and Fidelity</h2>
  <p class="card-subtitle">Fidelity — how well the extracted symbolic description matches the network's behavior, not its training fit</p>
</Band>

<Band tone="tan">
  <CoreDef>Knowledge extraction = deriving a readable symbolic description (rules, decision trees) from a neural network. Fidelity = agreement between that description and the network's actual behavior, not its fit to training data. LIME-family methods have very low fidelity — plausible-looking but causally disconnected explanations.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="fidelity-compare">
    <div class="fc-col high-fid">
      <span class="fc-head">High fidelity</span>
      <p>Student-teacher framework: rate at which extracted knowledge reproduces network predictions</p>
      <p>Provable soundness: formal guarantees on description precision</p>
    </div>
    <div class="fc-col low-fid">
      <span class="fc-head">Low fidelity (the problem)</span>
      <p>LIME's local linear approximation may diverge wildly from the original network on the same region</p>
      <p>Post-hoc explanation layers cannot guarantee faithful description of the underlying model</p>
    </div>
  </div>
  <div class="uses">
    <Kicker>Why extract knowledge</Kicker>
    <KV label="Bias discovery">GDPR compliance — reveal which proxies for protected variables the network actually uses</KV>
    <KV label="Model debugging">Find when the network relies on features it shouldn't (shortcut learning)</KV>
    <KV label="Neurosymbolic loop">Extracted symbolic knowledge feeds back into the next learning round as constraint</KV>
  </div>
  <div class="vs-mi">
    <span class="vsmi-label">vs mechanistic interpretability</span>
    <span class="vsmi-text">Knowledge extraction: map to symbolic rules. Mech interp: trace information flow at the activation level. Both demand causal accuracy</span>
  </div>
</Band>
