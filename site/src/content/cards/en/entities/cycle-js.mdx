<Band tone="tan">
  <CoreDef>Cycle.js treats the application as a pure function: it takes inputs from the outside world (sources, as observables) and returns outputs to the outside world (sinks, also observables). Every component is a sources → sinks pure function with the same signature as the whole application — the canonical fractal architecture: identical interface structure at every level, recursively composable.</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="arch-features">
    <Kicker>Architectural features</Kicker>
    <div class="af-row af-uniform">
      <span class="af-name">Uniform interface</span>
      <span class="af-desc">Every component shares the sources → sinks signature with the whole app — nesting at any depth preserves interface consistency</span>
    </div>
    <div class="af-row af-recursive">
      <span class="af-name">Recursive composition</span>
      <span class="af-desc">Components call other components like functions call other functions — fractal self-similar structure</span>
    </div>
    <div class="af-row af-driver">
      <span class="af-name">Side effects → driver layer</span>
      <span class="af-desc">Components are side-effect-free; side effects are pushed to the framework's driver layer — glue and logic are separated</span>
    </div>
  </div>
  <div class="agent-mapping">
    <Kicker>Structural isomorphism with agent systems</Kicker>
    <KV label="Implicit-loop correspondence">Cycle.js: sources → sinks pure function; Agent: context → actions, with the harness handling side effects</KV>
    <KV label="Constraints, not preset paths">Both shape behavior through constraints rather than preset paths — behavior emerges from interaction</KV>
    <KV label="Glue-logic separation">Driver layer ↔ harness layer — isomorphic design separating side effects (I/O) from reasoning (pure logic)</KV>
  </div>
</Band>
