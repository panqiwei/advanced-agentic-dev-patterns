</Band>

<Band tone="paper-raised" grow>
  <div class="five-dims">
    <Kicker>五维 Context Engineering 框架</Kicker>
    <div class="fd-row fd-offload">
      <span class="fd-name">Offloading</span>
      <span class="fd-desc">文件系统作为无限外部 memory，零 token 成本——把内容「移出」context</span>
    </div>
    <div class="fd-row fd-reduce">
      <span class="fd-name">Reduction</span>
      <span class="fd-desc">仅限可恢复压缩（保留路径/URL 指针）——防止信息永久丢失</span>
    </div>
    <div class="fd-row fd-retrieve">
      <span class="fd-name">Retrieval</span>
      <span class="fd-desc">文件搜索工具作为结构化检索层——按需加载而非全部载入</span>
    </div>
    <div class="fd-row fd-isolate">
      <span class="fd-name">Isolation</span>
      <span class="fd-desc">子 agent 在独立 context window 中运行——隔离避免干扰</span>
    </div>
    <div class="fd-row fd-cache">
      <span class="fd-name">Caching</span>
      <span class="fd-desc">Stable prefix + session affinity 路由——最大化 prefix cache 命中率</span>
    </div>
  </div>
  <div class="tech-decisions">
    <Kicker>代表性技术决策</Kicker>
    <KV label="Logit Masking 替代动态工具移除">修改 tools 数组破坏 KV cache，改用解码阶段 logit masking 约束动作空间</KV>
    <KV label="todo.md 注意力工程">每次迭代更新 todo.md，将全局计划维持在 context 近期位置——对抗「中间迷失」</KV>
    <KV label="错误保留策略">保留失败记录为模型提供隐式 belief update，错误恢复行为自然涌现</KV>
  </div>
