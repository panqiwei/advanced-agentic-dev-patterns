  <h2 class="card-title">长时 Agent 的 Harness 设计</h2>
  <p class="card-subtitle">当任务超出单个 context window——双 agent 架构 + 外部状态交接</p>
</Band>

<Band tone="tan">
  <CoreDef>两类失败模式同源：one-shotting（耗尽 context 留半成品）和 premature victory（误判已完成）。解法是把「一次性完成」改成「增量推进 + 状态交接」——Initializer 搭好脚手架，Coder 逐个 feature 推进。</CoreDef>
</Band>

<Band tone="paper-raised" cols={2} grow>
  <div class="agent">
    <div class="agent__label">① INITIALIZER</div>
    <div class="agent__role">一次性</div>
    <ul class="agent__list">
      <li>写 <span class="mono">init.sh</span>（开发环境启动）</li>
      <li>创建 <span class="mono">claude-progress.txt</span></li>
      <li>生成 JSON feature list（200+ 条）</li>
      <li>初始 git commit</li>
    </ul>
  </div>
  <div class="agent agent--coder">
    <div class="agent__label">② CODER</div>
    <div class="agent__role">每个 session</div>
    <ul class="agent__list">
      <li>读进度 + git log 恢复状态</li>
      <li>选一个 feature 端到端实现</li>
      <li>Puppeteer MCP 验证</li>
      <li>commit + 更新 <span class="mono">passes</span> 字段</li>
    </ul>
  </div>
</Band>

<Band tone="paper-sunk" dense>

  </div>
  <div class="handoff">

  </div>
