<Band tone="paper-raised" dense>
  <h2 class="card-title">分形架构</h2>
  <p class="card-subtitle">Fractal Architecture — 自相似性：任何子树是有效的树</p>
</Band>

<Band tone="tan">
  <CoreDef>系统中每个组件与整体具有相同结构和接口。可组合性不来自"组件很小"，而来自接口的一致性——接口同构时，组合结果仍满足相同的 API 契约，递归可组合。</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="four-rules">
    <Kicker>四条规则（Telesh 2016，归纳自 React / Angular / Elm / Cycle.js）</Kicker>
    <div class="rule-item">
      <span class="rule-n">①</span>
      <div>
        <span class="rule-name">统一接口</span>
        <span class="rule-desc">应用是由相同 API 的组件组成的树</span>
      </div>
    </div>
    <div class="rule-item">
      <span class="rule-n">②</span>
      <div>
        <span class="rule-name">递归组合</span>
        <span class="rule-desc">每个组件可以包含其他组件</span>
      </div>
    </div>
    <div class="rule-item">
      <span class="rule-n">③</span>
      <div>
        <span class="rule-name">无特权根节点</span>
        <span class="rule-desc">顶层组件与叶子组件没有本质区别</span>
      </div>
    </div>
    <div class="rule-item">
      <span class="rule-n">④</span>
      <div>
        <span class="rule-name">胶水与逻辑分离</span>
        <span class="rule-desc">引导/装配代码在组件体系之外完成；组件不知道自己在树中的位置</span>
      </div>
    </div>
  </div>
  <div class="agent-mapping">
    <KV label="Agent Skills 映射">每个技能是相同接口的目录（SKILL.md + 统一协议）— 规则①②</KV>
    <KV label="隐式循环呼应">Agent 不知道全局路径，行为由约束塑造 — 规则③④的结构同构</KV>
  </div>
</Band>
