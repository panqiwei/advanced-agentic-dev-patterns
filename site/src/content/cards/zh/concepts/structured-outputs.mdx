<Band tone="paper-raised" dense>
  <h2 class="card-title">结构化输出</h2>
  <p class="card-subtitle">Structured Outputs — 通过 JSON Schema + CFG 强制模型输出符合格式，但有隐性代价</p>
</Band>

<Band tone="tan">
  <CoreDef>结构化输出的实现链路：JSON Schema → 上下文无关文法（CFG）→ token 掩码（只允许当前合法的 token）。效果显著：OpenAI 数据显示 JSON mode 的格式错误率 11.97%，Structured Outputs 降至 0.1%。但隐性代价是"轨迹偏差"——强制格式会改变模型的推理路径，可能降低任务准确性。格式正确 ≠ 答案正确。</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="mode-compare">
    <Kicker>JSON mode vs Structured Outputs</Kicker>
    <div class="mc-row mc-head"><span>方式</span><span>格式错误率</span><span>机制</span><span>局限</span></div>
    <div class="mc-row"><span>JSON mode</span><span class="mc-bad">11.97%</span><span>提示词约束</span><span>模型自由发挥格式</span></div>
    <div class="mc-row mc-good"><span>Structured Outputs</span><span>0.1%</span><span>Schema→CFG token 掩码</span><span>轨迹偏差</span></div>
  </div>
  <div class="hidden-costs">
    <Kicker>隐性代价与工程原则</Kicker>
    <KV label="轨迹偏差">强制格式约束推理路径——某些任务中，自由文本的准确率反而更高</KV>
    <KV label="Anthropic grammar 缓存">固定 Schema 触发服务端缓存，重复使用同一 Schema 可降低延迟</KV>
    <KV label="渐进式结构化">先自由推理，最后一步用工具/结构化提取——分离推理与格式化</KV>
    <KV label="必要时才用">工具调用/API 对接才用 Structured Outputs；人类阅读输出不需要强制格式</KV>
  </div>
</Band>
