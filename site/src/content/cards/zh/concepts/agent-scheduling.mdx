<Band tone="tan">
  <CoreDef>当前框架无调度层——并发 agent 直接抢 GPU 显存，塞满报 OOM，释放后重试。这不是调度，是碰运气。AIOS 将 OS 进程调度类比引入 agent 资源管理。</CoreDef>
</Band>

<Band tone="paper-raised" grow>
  <div class="algo-compare">
    <div class="algo-col">
      <span class="algo-name">FIFO</span>
      <span class="algo-desc">先进先出</span>
      <ul class="algo-list">
        <li>按到达顺序处理</li>
        <li>简单可预测</li>
        <li>后到 agent 可能长等</li>
      </ul>
    </div>
    <div class="algo-divider"></div>
    <div class="algo-col">
      <span class="algo-name">Round Robin</span>
      <span class="algo-desc">时间片轮转</span>
      <ul class="algo-list">
        <li>每 agent 一个时间片</li>
        <li>更公平</li>
        <li>需 context snapshot/restore</li>
      </ul>
    </div>
  </div>
</Band>

<Band tone="paper-sunk" dense>
  <div class="mapping-section">
    <Kicker>与传统 OS 调度的映射</Kicker>
    <div class="mapping-grid">
      <span class="m-trad">进程</span><span class="m-arrow">↔</span><span class="m-agent">Agent 请求</span>
      <span class="m-trad">CPU 时间片</span><span class="m-arrow">↔</span><span class="m-agent">LLM 推理时间片</span>
      <span class="m-trad">上下文切换</span><span class="m-arrow">↔</span><span class="m-agent">Context snapshot/restore</span>
    </div>
    <p class="diff-note">关键差异：LLM 中断需保存生成中的 token 序列和搜索树，远比寄存器快照复杂</p>
  </div>
</Band>
